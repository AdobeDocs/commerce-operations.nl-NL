# ACSD-68499: De mutatie van GraphQL updateCartItems keert *succes* voor hoeveelheden voorbij beschikbare voorraad terug

Het ACSD-68499 flard bevestigt de kwestie waar de `updateCartItems` mutatie van GraphQL een onjuiste *succes* reactie wanneer het bijwerken van hoeveelheden terugkeert die beschikbare voorraad overschrijden, resulterend in opgepompte hoeveelheden en totalen. Deze patch is beschikbaar wanneer [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.74 wordt geÃ¯nstalleerd. De patch-id is ACSD-68499. Dit probleem wordt volgens de planning opgelost in Adobe Commerce 2.4.9.

## Betrokken producten en versies

**het flard wordt gecreeerd voor de versie van Adobe Commerce:**

* Adobe Commerce (alle implementatiemethoden) 2.4.8-p2

**Compatibel met de versies van Adobe Commerce:**

* Adobe Commerce (alle implementatiemethoden) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>De patch kan van toepassing worden op andere versies met nieuwe [!DNL Quality Patches Tool] versies. Om te controleren of de patch compatibel is met uw Adobe Commerce-versie, werkt u het `magento/quality-patches` -pakket bij naar de meest recente versie en controleert u de compatibiliteit op de [[!DNL Quality Patches Tool] : zoek naar patches op de pagina ](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html) . Gebruik de patch-id als een zoekwoord om de patch te zoeken.

## Probleem

De GraphQL `updateCartItems` mutatie in Adobe Commerce keert verkeerd *succes* reactie terug wanneer het bijwerken van hoeveelheden voorbij beschikbare voorraad, veroorzakend opgeblazen karttotalen en hoeveelheden.

<u> Stappen om </u> te reproduceren:

1. Maak een lege wagen.
1. Voeg een product met beperkte voorraad (bijvoorbeeld 10 beschikbare eenheden) toe aan het winkelwagentje.
1. Vraag het winkelwagentje op en noteer de `cart_item_id` .
1. Werk de hoeveelheid van het winkelwagentje bij tot 100 (overschrijdt de beschikbare voorraad van 10) met behulp van de volgende mutatie:

   ```
   mutation ($cart: String!, $cartItemId: Int!, $quantity: Float!) {
     updateCartItems(
       input: {
         cart_id: $cart
         cart_items: [{ cart_item_id: $cartItemId, quantity: $quantity }]
       }
     ) {
       cart {
         items {
           product {
             name
           }
           quantity
         }
         prices {
           grand_total {
             value
             currency
           }
         }
       }
     }
   }
   ```

1. Vraag het winkelwagentje opnieuw om het werkelijke aantal te controleren.

<u> Verwachte resultaten </u>:

Met voorraadvalidatie wordt voorkomen dat het aantal updates ongeldig wordt.

<u> Ware resultaten </u>:

De `updateCartItems` mutatie keert *succes* reactie met een hoeveelheid 100 terug.
